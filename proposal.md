# TheBakcstageProject

## What am I building and who is it for

I am building a minimalist, professional theatre management tool designed for directors and stage managers to coordinate productions. This isn't just a script reader. It’s a central dashboard for the technical side of the show, meant for anyone who needs to manage props, timing, and cues without the usual mess of papers and spreadsheets.

## Why?
Managing a production is chaotic, especially when you are balancing acting and Directing at the same time. I want to build something that actually helps me stay organized during rehearsals. Instead of jumping between different files, I need one place where I can see the stage layout, track my cues, and keep an eye on the production timeline so the show runs smoothly.

## MVP vs. Stretch Goals
The core of the app will focus on three main things: a digital cue sheet that highlights what’s happening in real-time, a visual prop manager where I can map out the stage layout for different scenes, and a production clock to track scene durations. For stretch goals, I want to add a cast contact directory and maybe a small section for director's notes that I can save for later.

## What technologies do I plan to use?
I’m keeping it clean with a minimalist black, white, and grey palette using HTML and CSS. I’ll use a CSS grid to build the stage map for the prop manager. The heavy lifting will be done with JavaScript—using event listeners for the cue triggers, DOM manipulation to move props around the stage, and local storage so that all my scene layouts and notes actually stay there when I close the tab.
